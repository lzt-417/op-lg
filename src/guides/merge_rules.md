# Merge Rules — 审查报告合并规则

> **用途：** COO 合并 3 份 Pass 报告为综合修复指令的规则。
> **输入：** logic-review + adversarial-review + prose-review
> **输出：** editor-review（综合修复指令）

---

## 合并逻辑

| 来源 | P0 必修 | P1 应修 | P2 建议 |
|------|---------|---------|---------|
| **Pass 1 逻辑审查** | 全部 P0 项 | 全部 P1 项 | 无（逻辑层无 P2） |
| **Pass 2 对抗编辑** | 无 | OVER-EXPLAIN(CUT) + REDUNDANT(REWRITE) | FILLER / TELL_NOT_SHOW / REPEATED_PHRASE / 总结性反思 |
| **Pass 3 文笔审查** | 无 | 评分 <6 的维度 | 评分 ≥6 但可优化的维度 / 禁用词 |

### 具体规则

1. **P0 只来自 Pass 1。** 逻辑层的 P0 问题是硬性矛盾，必须修。Pass 2 和 Pass 3 不产生 P0。

2. **Pass 2 的 CUT 和 REWRITE 建议归入 P1。** 这些是量化确认的冗余，修复成本明确。但需遵守 adversarial-edit-guide.md 中的保护规则（PROTECTED 段落不纳入）。

3. **Pass 3 中评分 <6 的维度归入 P1。** 这些是文笔质量明显不达标的部分。

4. **Pass 2 和 Pass 3 的其余建议归入 P2。** 这些是可优化但不紧急的问题。

5. **同位置多重问题合并。** 如果同一位置同时被多个 Pass 标记问题，合并为一条，标注来源。
